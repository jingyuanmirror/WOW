# 作者介绍页产品需求文档 (PRD)

## 1. 页面概述

### 1.1 页面定位
作者介绍页是品牌官网的重要内容页，旨在展示设计师/作者的个人形象、专业背景、设计理念及其影响力，增强用户信任感和品牌背书。通过丰富的内容和互动入口，促进用户对作者的深入了解和社区互动。

### 1.2 页面目标
- 展示作者的专业形象与成就
- 传递作者的设计理念与品牌价值观
- 引导用户关注作者的自媒体账号，拓展品牌影响力
- 增强用户信任感和社区归属感

---

## 2. 页面结构与功能模块

### 2.1 作者基础信息区
- **头像**：高清头像，支持圆形/特殊形状遮罩，尺寸建议 200x200px
- **姓名/昵称**：突出显示，24-32px 粗体
- **头衔/身份**：如“魔兽世界UI设计师”、“独立开发者”等
- **一句话简介**：简明扼要，突出专业性或个人特色

### 2.2 详细个人简介区
- **多段落介绍**：
  - 第一段：自我介绍（如从业经历、专业背景）
  - 第二段：设计理念（对UI/产品的理解与追求）
  - 第三段：成就与愿景（获奖、用户数、未来目标等）
- **排版**：每段 2-4 行，易读性强

### 2.3 设计理念展示区
- **区块标题**：如“我的设计理念”
- **理念内容**：可配合图标/插画，突出关键词（如“美观”、“实用”、“高效”、“创新”）
- **引用样式**：可用引号或高亮背景

### 2.4 自媒体账号区
- **区块标题**：如“关注我的自媒体”
- **账号列表**：
  - 支持多平台（如 Twitter/X、Discord、Bilibili、YouTube、NGA、知乎、微信公众号等）
  - 每个账号包含：
    - 平台图标（建议使用 Lucide 或品牌色）
    - 账号名称/昵称
    - 跳转链接（新窗口打开）
    - 可选：粉丝数/认证标识
- **布局**：横向或网格排列，移动端自动换行
- **交互**：悬停放大/高亮，点击跳转

### 2.5 数据与成就区
- **关键数据指标**：
  - 皮肤作品数量
  - 活跃用户数
  - 平均评分
  - 社区奖项/荣誉
- **展示方式**：大数字+图标+描述，支持数字递增动画
- **布局**：2x2 网格或横向排列

### 2.6 代表作品/精选案例区（可选）
- **展示作者代表性皮肤或项目**
- **每个作品包含**：封面图、名称、简要描述、跳转详情
- **布局**：卡片式，3-4个为宜

### 2.7 联系方式区（可选）
- **邮箱/表单**：便于用户或合作方联系
- **隐私提示**：不会公开展示邮箱

### 2.8 SEO与无障碍
- 语义化HTML结构，合理使用`<section>`、`<header>`、`<main>`等标签
- 所有图片需提供alt文本
- 颜色对比度符合WCAG 2.1 AA
- 所有外链加`rel="noopener noreferrer"`

---

## 3. 响应式设计
- 桌面端：左右分栏（左：头像+社交，右：简介/理念/成就）
- 平板/移动端：纵向堆叠，头像/社交在上，内容在下
- 所有区块自适应宽度，按钮/图标触摸友好

---

## 4. 技术实现建议
- **组件结构**：
  - `src/components/author-profile.tsx`（主组件）
  - `src/components/social-links.tsx`（自媒体账号）
  - `src/components/author-stats.tsx`（数据成就）
- **样式**：Tailwind CSS v4，金色/品牌色点缀
- **图标**：Lucide React
- **动画**：Framer Motion（数字递增、悬停放大）
- **数据**：静态/服务端获取均可，便于后续动态化

---

## 5. 验收标准
- 所有核心信息完整展示
- 所有自媒体账号可跳转，图标清晰
- 响应式适配良好
- 视觉风格与品牌一致
- 代码通过ESLint和TypeScript检查
- 无console报错，性能良好
